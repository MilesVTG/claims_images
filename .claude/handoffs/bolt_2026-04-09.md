## Gotchas / Known Issues

1. **Route ordering matters:** In both claims.py and photos.py, parameterized routes (`/{x}/{y}`) must come AFTER fixed-prefix routes (`/status/...`, `/serve/...`) to avoid path capture conflicts.

2. **Alembic migration 003 not yet run:** The `error_logs` table migration exists but hasn't been applied to any database yet. Run `alembic upgrade head` before testing error logging.

3. **Worker functions removed:** `process_single_photo()` and `run_claim_analysis()` were deleted — their logic is now inlined in `handle_pubsub_push()`. If any tests reference these functions directly, they'll need updating.

4. **Error middleware is duplicated:** API and Worker each have their own copy of `error_logging.py` since they're separate services with separate `app/` packages. If the shared logic needs changes, update both.

5. **cloud_vision and gemini_analysis stages don't re-raise:** They log the error but gracefully degrade (Vision returns `{"error": ...}`, Gemini returns a fallback result). This matches the original behavior.

6. **No auth on /api/errors:** Intentional — scripts and Claude Code need direct access. Not for production without adding auth.
